## Git Commit Rules
When generating git commit messages, you MUST follow this format:
`[Type] Subject`

- **Types**: [Feat], [Fix], [Refactor], [Style], [Docs], [Test], [Chore], [Design]
- **Subject**: Write in Korean. Concise summary. No period at the end. Use imperative mood.
- **Example**: 
  - `[Feat] imply JWT `
  - `[Fix] resolve error for rendering chart in dashboard`
